# Настройка XRay (Hiphosting)

Конфигурация XRay-панели (на базе 3X-UI) на хостинге Hiphosting. В данном разделе задокументированы параметры подключения и маршрутизации, которые используются для перенаправления трафика через Cloudflare WARP (протокол Wireguard). Это позволяет скрыть реальный IP-адрес сервера и обходить региональные блокировки, которые могут детектировать IP хостинга.

## ⚙️ Настройка Outbound (WARP через Wireguard)

Вместо сырого JSON-конфига, ниже представлена таблица с параметрами исходящего соединения (Outbound), настроенного в панели.

| Параметр в панели (JSON) | Значение                              | Описание и влияние                                                                                                                         |
| :----------------------- | :------------------------------------ | :----------------------------------------------------------------------------------------------------------------------------------------- |
| **Protocol**             | `wireguard`                           | Используемый протокол для исходящего соединения. Позволяет туннелировать трафик через сеть Cloudflare WARP.                                |
| **Tag**                  | `warp`                                | Уникальный идентификатор исходящего канала. Используется в правилах маршрутизации для направления трафика именно в этот туннель.           |
| **MTU**                  | `1280`                                | Maximum Transmission Unit. Значение 1280 оптимально для Wireguard/WARP, чтобы избежать фрагментации пакетов и потери скорости.             |
| **SecretKey**            | `+Jeo4SWzuQfalmb...`                  | Приватный ключ Wireguard-клиента (вашего сервера). Генерируется при регистрации в WARP.                                                    |
| **Address**              | `172.16.0.2/32`<br>`2606:4700.../128` | Внутренние IPv4 и IPv6 адреса, выданные вашему серверу в сети Cloudflare WARP.                                                             |
| **DomainStrategy**       | `ForceIPv4v6`                         | Стратегия разрешения DNS. Заставляет XRay резолвить и использовать как IPv4, так и IPv6 адреса для соединений.                             |
| **Reserved**             | `[125, 34, 79]`                       | Зарезервированные байты ClientID для обхода блокировок WARP в РФ. Это специфичная для WARP фича, позволяющая маскировать Wireguard-трафик. |
| **Peers / PublicKey**    | `bmXOC+F1FxEM...`                     | Публичный ключ сервера Cloudflare WARP. Необходим для шифрования трафика в сторону Cloudflare.                                             |
| **Peers / Endpoint**     | `engage.cloudflareclient.com:2408`    | Адрес и порт сервера Cloudflare, к которому подключается туннель.                                                                          |
| **Peers / AllowedIPs**   | `0.0.0.0/0`, `::/0`                   | Указывает, какие IP-адреса могут маршрутизироваться через этот пир (в данном случае — весь интернет-трафик).                               |
| **NoKernelTun**          | `false`                               | Использовать ли userspace-реализацию (XRay) вместо модуля ядра ОС. `false` означает попытку использования TUN-интерфейса, если доступно.   |

## 🔀 Правила маршрутизации (Routing)

Правила, которые определяют, какой трафик куда пойдет. Они обрабатываются сверху вниз:

| Правило (Type) | Условие | Куда направляется (Outbound Tag) | Назначение |
| :--- | :--- | :--- | :--- |
| `field` | Inbound: `api` | `api` | Локальный API-трафик самой панели (статистика, управление) остается внутри. |
| `field` | IP: `geoip:private` | `blocked` (blackhole) | Блокировка доступа к локальным/приватным IP адресам извне. Защищает сервер от попыток сканирования внутренней сети через ваш прокси. |
| `field` | Protocol: `bittorrent` | `blocked` (blackhole) | Блокировка торрент-трафика. Защищает сервер от абуз и жалоб правообладателей (DMCA), которые часто приходят на хостинги. |
| `field` | Network: `TCP,UDP` | `warp` | **Основное правило:** Весь остальной TCP и UDP трафик от клиентов направляется в туннель Cloudflare WARP. |
