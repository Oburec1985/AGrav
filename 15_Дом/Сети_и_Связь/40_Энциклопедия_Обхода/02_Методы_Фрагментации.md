# 🧪 Теория фрагментации и Desync

Фрагментация трафика является ключевым методом обхода систем DPI, основанным на манипуляции структурой TCP/UDP пакетов для сокрытия сигнатур протоколов.

## Основные концепции

### 1. Разрезание пакетов (Split)
Метод, при котором полезная нагрузка (payload) разделяется на части. Например, если DPI ищет строку `youtube.com` в первом пакете, разделение этой строки между двумя пакетами (`yout` + `ube.com`) сделает обнаружение невозможным без сборки потока в памяти (что ресурсозатратно для провайдера).

**Параметры (на примере Zapret):**
- `--dpi-desync-split-pos`: Точка разреза. Вариант `2` или `sld` (second level domain) часто используется для TLS ClientHello.

### 2. Мульти-фрагментация (Multisplit)
Усложненная версия, при которой пакет делится на множество мелких фрагментов, что максимально нагружает буферы DPI-систем.

### 3. Перекрытие сегментов (Sequence Overlap)
Метод `seqovl`, при котором данные в двух последовательных пакетах перекрываются. Реальный сервер корректно соберет данные по порядковым номерам (Sequence Numbers), в то время как DPI может ошибиться при реконструкции.

## Стратегии десинхронизации (Desync Strategies)

- **hostfakesplit**: Подмена оригинального заголовка Host в HTTP-запросе на фейковый, с последующим разрезанием пакета.
- **disorder**: Изменение порядка следования пакетов. Сервер восстановит порядок, но DPI может не успеть проанализировать "хаотичный" поток.
- **fake**: Отправка полностью поддельного пакета с TTL, достаточным только для достижения DPI провайдера. Последующий легитимный пакет "проскакивает" под защитой фейка.

## Резюме параметров

| Параметр | Описание |
| :--- | :--- |
| `badsum` | Инъекция пакетов с намеренно неверной контрольной суммой. |
| `badseq` | Использование некорректных порядковых номеров для запутывания DPI. |
| `foling` | Общее название техник обмана систем анализа. |
