# 🚀 Zeroblock: Автономный обход

**ZeroBlock** — это система обхода блокировок следующего поколения, разрабатываемая как эволюционная замена скриптовым решениям. Основной упор сделан на автономность, производительность и удобство управления через "Мастер настроек".

## Технологический стек

ZeroBlock значительно отличается от предшественников (Podkop/Zapret) архитектурным подходом:
- **Native C Core**: Перевод логики с тяжелых bash-скриптов на нативный C-код. Это снижает нагрузку на CPU роутера и ускоряет время старта.
- **Netlink (libmnl)**: Прямое взаимодействие с ядром для управления маршрутами и правилами, минимизируя вызовы внешних утилит типа `ip route`.
- **Sing-box Backend**: Использует Sing-box в качестве ядра проксирования, разделяя конфигурации на микро-модули в `/tmp/zeroblock/conf.d`.

## Ключевые функции

1. **Подписки (Subscriptions)**:
   - **URLTest**: Автоматический выбор сервера с наименьшей задержкой (авто-выбор "лучшего").
   - **Selector**: Ручное переключение между прокси в реальном времени через Dashboard.
2. **BitTorrent Offload**: 
   - Обнаружение торрент-трафика по характерным паттернам и его автоматический перевод в статус `Direct` (в обход прокси-ядра). 
   - **Логика**: При обнаружении BitTorrent-пакета соединение помечается меткой offload. 
   - **Таймаут**: Используется системный таймаут в 60 секунд. Если в течение этого времени торрент-активность прекращается, правила очищаются. Это критично для стабильности RAM и минимизации нагрузки на CPU роутера.
3. **BadWAN Monitor**:
   - Автоматическая проверка доступности ресурсов и восстановление работоспособности туннелей без участия пользователя.
4. **Proxy Router Traffic**:
   - Опция, позволяющая направлять трафик самого роутера (например, `opkg update`) через настроенные прокси.

## Диагностика и отладка

Для ZeroBlock критически важны логи уровня **Trace**. В системе реализован "Модуль диагностики", который генерирует подробные отчеты, скрывая при этом приватную информацию (ключи, полные URL подписок).
