# 🔬 Природа DPI и методы деградации

Технология **DPI (Deep Packet Inspection)** — это метод глубокого анализа трафика, используемый интернет-провайдерами для идентификации, фильтрации и модификации пакетов данных на основе их содержимого (L7 уровень OSI), а не только заголовков IP/TCP.

## Механизмы обнаружения

1. **Protocol Fingerprinting**: Идентификация протоколов (VLESS, Shadowsocks, QUIC) по специфическим паттернам байтов.
2. **SNI Analysis**: Проверка поля Server Name Indication в TLS ClientHello для выявления запрещенных доменных имен.
3. **Entropy Analysis**: Измерение "случайности" данных. Высокая энтропия характерна для зашифрованного трафика, что может служить триггером для блокировки (например, при использовании "чистого" Shadowsocks).
4. **Flow Analysis**: Анализ временных интервалов и размеров пакетов (размер видео-фреймов YouTube отличается от обычного веб-серфинга).

## Методы противодействия (Degradation)

Для нейтрализации DPI применяются стратегии "десинхронизации" и "запутывания" (Obfuscation):

### 1. Десинхронизация (Desync)
Внедрение в поток TCP-пакетов, которые выглядят валидными для DPI-системы, но игнорируются целевым сервером. Это заставляет DPI "потерять" контекст сессии.
- **Bad Checksum**: Пакеты с неверной контрольной суммой.
- **Bad Sequence**: Пакеты с некорректными порядковыми номерами.

### 2. Подмена SNI (SNI Spoofing/Faking)
Модификация или сокрытие имени сервера в процессе установки TLS-соединения.
- **ClientHello Fragmentation**: Разрезание TLS-пакета таким образом, чтобы поле SNI было разделено между двумя TCP-сегментами.
- **Domain Fronting**: Использование легитимного домена (например, Google) в качестве "витрины" для обращения к заблокированному ресурсу.

### 3. TTL Manipulation (Ограничение дальности)
Установка значения `Time to Live` таким образом, чтобы "фейковый" пакет дошел до DPI провайдера (обычно 3-5 хопов), но был отброшен следующим маршрутизатором, не дойдя до целевого сервера.
