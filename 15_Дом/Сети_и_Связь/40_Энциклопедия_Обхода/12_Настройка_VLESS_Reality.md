# Настройка VLESS + Reality (3X-UI)

Инструкция по развертыванию аналогичного соединения на базе протокола VLESS с маскировкой Reality.

## 📖 Как читать эту таблицу?

Параметры разделены на три логических слоя. Понимание этих слоев поможет вам быстро адаптировать конфиг под любой сервер:

1.  **⚙️ Скелет (База)**: `Protocol`, `Port`, `Transmission`. Это "труба", по которой текут данные. Если они не совпадают на клиенте и сервере, связи не будет.
2.  **🎭 Маскировка (Reality)**: `Security`, `Dest`, `SNI`, `Fingerprint`. Это ваша "маскировка". Она заставляет ваш трафик выглядеть как обычный просмотр сайта (например, Google или Thomann).
3.  **🔑 Доступ (Безопасность)**: `UUID`, `Short ID`, `Keys`. Это ваши ключи от двери. Только тот, у кого есть правильный ID и публичный ключ, сможет войти.

| Параметр в 3X-UI | Значение (Пример) | Описание и влияние |
| :--- | :--- | :--- |
| **Protocol** | `vless` | Протокол передачи данных. Не имеет встроенного шифрования, поэтому требует TLS/Reality. Самый быстрый и современный. |
| **Port** | `4433` | Порт, который слушает сервер. Влияет на доступность: порты 443/80 сложнее заблокировать, но они могут быть заняты веб-сервером. |
| **Transmission** | `tcp` | Способ передачи пакетов. TCP — стандарт, надежен. UDP (через gRPC/H2) может быть быстрее, но чаще блокируется DPI. |
| **Security** | `reality` | Тип "брони". Reality подменяет отпечаток TLS сервера на отпечаток реального разрешенного сайта. |
| **Dest** | `google.com:443` | Сайт-мишень для маскировки. Если DPI зайдет на ваш IP, сервер перенаправит его сюда, прикидываясь прокси-сервером этого сайта. |
| **SNI** | `google.com` | Имя сервера в заголовке пакета. Должно совпадать с доменом в Dest. Именно его видит провайдер. |
| **Short ID (sid)** | `1fca` | Дополнительный ключ безопасности. Позволяет серверу отличать "своих" клиентов от случайных сканеров. |
| **SpiderX (spx)** | `/` | Путь, по которому "паук" маскировки будет сканировать целевой сайт. Обычного `/` достаточно. |
| **Private Key** | *(генерируется)* | Секретный ключ сервера. Никогда не передается клиенту. |
| **Public Key (pbk)** | *(из пары)* | Публичный ключ. Передается клиенту в ссылке, используется для установления защищенного соединения. |
| **User ID (UUID)** | `uuid-string` | Уникальный ключ пользователя. Влияет на авторизацию: сервер пустит только того, чей UUID прописан в конфиге. |
| **Fingerprint (fp)** | `chrome` | Имитация поведения конкретного браузера. Помогает обходить проверку на "робота/прокси" со стороны продвинутых DPI. |

## Настройка выхода через WARP (Cloudflare)

Для того чтобы трафик с конкретного соединения выходил в интернет через IP Cloudflare (WARP):

1. **Тегирование**: В настройках Inbound установите уникальный **Tag** (например, `oburec-in`).
2. **Проверка Outbound**: Убедитесь, что в `Xray Configuration -> Outbounds` есть секция с протоколом `wireguard` и тегом `warp`.
3. **Маршрутизация (Routing)**:
   - В 3X-UI теги для инбаундов часто генерируются автоматически: `inbound-ПОРТ` (например, `inbound-4433`).
   - Добавьте правило: `Inbound Tag` = `inbound-4433` (или ваш тег) -> `Outbound Tag` = `warp`.
   - Это заставит весь трафик, приходящий на этот Reality-порт, уходить в интернет через Cloudflare WARP.
4. **Применение**: Сохраните настройки и нажмите **Restart Xray**.

## 📊 Как читать таблицу параметров
При импорте конфига в клиент (например, v2rayN или Nekobox) вы увидите таблицу. Вот что значат основные поля:

| Параметр | Значение | На что влияет |
| :--- | :--- | :--- |
| **Address** | IP сервера | Куда стучится клиент. |
| **Port** | 4433 (или др.) | Порт входа. Должен быть открыт в Firewall. |
| **SNI** | thomannmusic.com | Маскировка. Сайт, под который мы "прикидываемся". |
| **Flow** | xtls-rprx-vision | Алгоритм шифрования и защиты от активного детектирования. |
| **Security** | reality | Режим невидимости. |
| **Fingerprint** | chrome | Отпечаток браузера, чтобы трафик выглядел как обычный серфинг. |

## 🛠 Проверка работоспособности
Чтобы убедиться, что WARP работает через прокси, выполните команду внутри SSH сервера:
```bash
curl --socks5-hostname 127.0.0.1:4433 https://www.cloudflare.com/cdn-cgi/trace
```
*Если в ответе `warp=on` — связка Reality+WARP настроена верно.*
