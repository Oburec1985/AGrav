# 🌀 Экосистема Zapret

**Zapret** — это мощный программный комплекс для обхода систем DPI на уровне пакетов, не требующий использования внешних прокси-серверов или VPN (хотя поддерживающий совместную работу с ними). Основная идея заключается в модификации трафика таким образом, чтобы DPI провайдера не смог его распознать или заблокировать.

## Основные компоненты

### 1. NFQWS (Netfilter Queue Web Surgeon)
Работает на уровне L3/L4 (сетевой/транспортный уровень). Перехватывает пакеты через `nfqueue` и применяет к ним правила десинхронизации (фрагментация, фейк-пакеты).
- Наиболее эффективен для обхода блокировок по IP/порту.
- Работает прозрачно для приложений.

### 2. TPWS (Transparent Proxy Web Surgeon)
Работает на уровне L7 (прикладной уровень). Это прозрачный TCP-прокси, который модифицирует HTTP/HTTPS запросы (например, изменяет регистр заголовка Host или вставляет пробелы).
- Эффективен против простых DPI, анализирующих только HTTP-заголовки.

## Ключевые стратегии (Commands)

В экосистеме Zapret стратегия описывается набором флагов. Пример экспертной конфигурации для YouTube:
`--dpi-desync=fake,multisplit --dpi-desync-split-pos=2 --dpi-desync-fake-tls=0x0F0F0F0F`

### Описание флагов:
- `--dpi-desync=fake`: Посылка фейкового пакета перед основным.
- `--dpi-desync-ttl`: Установка TTL для фейкового пакета, чтобы он умер сразу за DPI провайдера.
- `--dpi-desync-fooling`: Техники обмана (`badsum`, `badseq`, `md5sig`).
- `--dpi-desync-repeats`: Количество повторов фейковых пакетов (часто ставится 6-10 для надежности).

## Совместимость и конфликты

Zapret может конфликтовать с системами точечной маршрутизации (Podkop/Zeroblock). 
**Правило безопасного использования:**
1. Разделять домены: если домен в списке Zapret, его не должно быть в Podkop.
2. Маркировка трафика: использовать разные метки (`fwmark`). В прошивках RouteRich Zapret часто использует метку `0x2`, а прокси — `0x40000000`.
