# 🔧 Установка WARP на VPS с Amnezia

Эта инструкция создана для того, чтобы максимально просто и понятно объяснить процесс настройки сервера (VPS) с установленной AmneziaWG. Наша цель — завернуть весь интернет-трафик клиентов VPN в туннель Cloudflare WARP. Это поможет обходить блокировки, маскировать трафик и повысить стабильность соединения.

Инструкция написана "для школьника": мы разберем каждый шаг, объясним, зачем мы вводим ту или иную команду, чтобы ты понимал, что именно происходит на твоем сервере.

---

## Подготовка
Для выполнения этой инструкции тебе понадобится:
1. Арендованный VPS сервер с установленной ОС Linux (рекомендуется Ubuntu/Debian).
2. На сервере уже должен быть установлен и работать VPN от Amnezia (AmneziaWG).
3. Доступ к серверу по SSH (IP-адрес, логин `root` и пароль).

Для выполнения команд просто копируй их и вставляй в терминал (консоль) сервера, после чего нажимай Enter.

---

## Шаг 1: Включение IP-форвардинга (пересылки пакетов)

**Зачем это нужно?** 
По умолчанию Linux-сервер забирает интернет-пакеты себе и никуда не отдает. Чтобы сервер работал как роутер (передавал интернет от твоего телефона через VPN дальше в сеть WARP), нам нужно включить эту функцию.

**Что делать:**
1. Для начала проверим, включена ли эта функция:
   ```bash
   sysctl net.ipv4.ip_forward
   ```
   *Если в ответ выдаст `net.ipv4.ip_forward = 1`, значит все отлично, переходи к Шагу 2.*
   *Если выдаст `0`, выполняем следующие две команды.*

2. Включаем пересылку пакетов навсегда:
   ```bash
   echo "net.ipv4.ip_forward = 1" > /etc/sysctl.d/99-warp-forward.conf
   sysctl -p /etc/sysctl.d/99-warp-forward.conf
   ```

## Шаг 2: Установка WireGuard и программы wgcf

**Зачем это нужно?**
WARP от Cloudflare работает на базе протокола WireGuard. Чтобы не настраивать всё руками, мы скачаем маленькую программу-помощник `wgcf`. Она сама зарегистрирует нам аккаунт в Cloudflare WARP и создаст готовый файл настроек.

**Что делать:**
1. Скачиваем и устанавливаем нужные системные пакеты (программы для работы сети и скачивания файлов). Команда длинная, но она всё сделает автоматически, без лишних вопросов:
   ```bash
   apt-get update && DEBIAN_FRONTEND=noninteractive apt-get install -y wireguard-tools iptables iproute2 curl wget
   ```

2. Скачиваем саму программу `wgcf` на сервер:
   ```bash
   wget -O /usr/local/bin/wgcf https://github.com/ViRb3/wgcf/releases/download/v2.2.22/wgcf_2.2.22_linux_amd64
   ```

3. Разрешаем серверу запускать эту программу:
   ```bash
   chmod +x /usr/local/bin/wgcf
   ```

---

## Шаг 3: Настройка и создание профиля WARP

**Зачем это нужно?**
Теперь с помощью установленной программы мы получим доступ к сети WARP. Мы создадим конфиг (настройки) и скажем серверу: "Сделай для WARP отдельный маршрут (Таблица 111)". Это нужно, чтобы WARP не забрал под себя ВЕСЬ интернет на сервере и не отрезал тебя от управления им.

**Что делать:**
1. Создаем папку для работы и заходим в нее:
   ```bash
   mkdir -p /root/warp_setup && cd /root/warp_setup
   ```

2. Регистрируем аккаунт в системе Cloudflare WARP (флаг `--accept-tos` означает, что мы автоматически соглашаемся с правилами использования):
   ```bash
   wgcf register --accept-tos
   ```

3. Создаем файл настроек (генерируем профиль):
   ```bash
   wgcf generate
   ```

4. Копируем готовый файл настроек туда, где его найдет WireGuard:
   ```bash
   cp wgcf-profile.conf /etc/wireguard/warp0.conf
   ```

5. Добавляем в настройки магическую строчку про отдельную таблицу маршрутизации (чтобы трафик шел правильно):
   ```bash
   sed -i '/\[Interface\]/a Table = 111' /etc/wireguard/warp0.conf
   ```

---

## Шаг 4: Запуск туннеля WARP

**Зачем это нужно?**
Настройки готовы, теперь нужно просто включить туннель и сделать так, чтобы он сам включался, если сервер вдруг перезагрузится.

**Что делать:**
1. Поднимаем (запускаем) интерфейс WARP:
   ```bash
   wg-quick up warp0
   ```

2. Добавляем его в автозагрузку системы:
   ```bash
   systemctl enable wg-quick@warp0
   ```

---

## Шаг 5: Настройка правил маршрутизации (САМЫЙ ВАЖНЫЙ ШАГ!)

**Зачем это нужно?**
Если просто направить весь трафик в WARP, то произойдет "асимметричная маршрутизация". Телефон пошлет запрос на сервер, сервер отправит ответ... но отправит его не обратно телефону, а в туннель WARP! В итоге на телефоне VPN покажет "подключено", но интернет работать не будет. Нам нужно добавить исключения: служебный трафик сервера идет напрямую, а пользовательский интернет — через WARP.

**Что делать:**
1. **Узнай порт своей AmneziaWG.** 
   Его можно посмотреть в самом приложении Amnezia на телефоне/компьютере (в настройках подключения) или введя на сервере команду `wg show`. Запомни эти 4-5 цифр (например, 51820 или 32451).

2. **Добавляем исключения.** Вводим следующие команды по очереди, чтобы "оставить в покое" служебный трафик.
   *Для внутренней сети Docker:*
   ```bash
   ip rule add to 172.29.172.0/24 table main priority 5
   ```
   *Для локальной сети самого VPN:*
   ```bash
   ip rule add to 10.8.1.0/24 table main priority 6
   ```
   *Для порта VPN (обязательно замени `<ТВОЙ_ПОРТ>` на цифры из пункта 1!)*:
   ```bash
   ip rule add sport <ТВОЙ_ПОРТ> table main priority 7
   ```

3. **Заворачиваем пользователей в WARP.** Теперь говорим, что все клиенты Amnezia (которые сидят на интерфейсе `amn0`) должны выходить в интернет через нашу 111-ю таблицу (через WARP):
   ```bash
   ip rule add iif amn0 table 111 priority 8
   ```
    *(Примечание: `amn0` — это стандартное имя сети, если Amnezia установлена в Docker. Если ты ставил её иначе, интерфейс может называться `awg0`).*

    > [!TIP]
    > Чтобы правила не пропадали после перезагрузки сервера, их нужно добавить прямо в файл `/etc/wireguard/warp0.conf` в секцию `[Interface]` вот так:
    > ```ini
    > PostUp = ip rule add to 172.29.172.0/24 table main priority 5
    > PostUp = ip rule add to 10.8.1.0/24 table main priority 6
    > PostUp = ip rule add sport <ТВОЙ_ПОРТ> table main priority 7
    > PostUp = ip rule add iif amn0 table 111 priority 8
    > PostDown = ip rule del to 172.29.172.0/24 table main priority 5
    > PostDown = ip rule del to 10.8.1.0/24 table main priority 6
    > PostDown = ip rule del sport <ТВОЙ_ПОРТ> table main priority 7
    > PostDown = ip rule del iif amn0 table 111 priority 8
    > ```

4. **Настраиваем NAT (Маскарадинг).** Это нужно, чтобы интернет думал, что твои запросы исходят от самого интерфейса WARP:
   ```bash
   iptables -t nat -A POSTROUTING -o warp0 -j MASQUERADE
   ```

---

## Шаг 6: Сохранение настроек и перезапуск

**Зачем это нужно?**
Все правила, которые мы написали в Шаге 5, работают только до перезагрузки сервера. Нам нужно их зафиксировать. А затем — перезапустить саму AmneziaWG, чтобы всё заработало вместе.

**Что делать:**
1. Устанавливаем утилиту для сохранения правил и сразу сохраняем их. Команды длинные, но они всё делают автоматически:
   ```bash
   echo iptables-persistent iptables-persistent/autosave_v4 boolean true | debconf-set-selections
   echo iptables-persistent iptables-persistent/autosave_v6 boolean true | debconf-set-selections
   apt-get install -y iptables-persistent && netfilter-persistent save
   ```

2. Перезапускаем контейнер AmneziaWG (если она установлена по умолчанию через официальное приложение):
   ```bash
   docker restart amnezia-awg2
   ```
   *(Если Amnezia устанавливалась без Docker напрямую в систему, команда будет такой: `systemctl restart awg-quick@awg0`)*

3. **Готово!** Можно проверить статус правил командой `ip rule` и статус туннелей командой `wg show`. Теперь на твоем телефоне должен мгновенно подключаться VPN и стабильно работать интернет!
