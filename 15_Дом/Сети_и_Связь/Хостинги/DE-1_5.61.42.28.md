## 🚀 Глобальная маршрутизация (WARP)

В отличие от других серверов, здесь настроено принудительное перенаправление всего трафика в WARP.

### 1. Настройка Outbound (Исходящий канал)
```json
{
  "tag": "warp",
  "protocol": "wireguard",
  "settings": {
    "secretKey": "REDACTED_PRIVATE_KEY",
    "address": ["172.16.0.2/32", "2606:4700:110:8731:2276:1ad:79a5:f366/128"],
    "peers": [
      {
        "publicKey": "bmXOC+F1FxEMF9dyiK2H5/1SUtzH0JuVo51h2wPfgyo=",
        "endpoint": "162.159.192.1:2408"
      }
    ],
    "reserved": [81, 12, 175],
    "mtu": 1280
  }
}
```

### 2. Правила маршрутизации (Routing Rules)
Чтобы WARP заработал для всех, в панели добавлены следующие правила:

| Priority | Type | Port / Network | Outbound Tag | Описание |
| :--- | :--- | :--- | :--- | :--- |
| 1 | Field | `geosite:google` | `warp` | Весь Google-трафик через WARP. |
| 2 | Field | `network: tcp,udp` | `warp` | **Глобальное правило**: весь интернет-трафик уходит в WARP. |
