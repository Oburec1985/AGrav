# Заметки по протоколу zad_koman

## Наблюдения
1. Проект написан на VB6.
2. Обмен "грязный" — через глобальный буфер обмена. Это может мешать пользователю (перезаписывать его данные), но для системы имитации в выделенной среде допустимо.
3. Конец пакета `\` важен для парсера.
4. Отрицательные статусы (например `-27`) — ключевой механизм синхронизации: Пульт ждет именно их.
5. Данные измерений передаются как плоский список чисел через запятую, которые нужно интерпретировать как пары Re/Im.

## Риски
- Если в буфер попадет посторонний текст, парсер `Val()` может выдать 0 или ошибку (хотя в коде есть `On Error Resume Next` в некоторых местах).
- При больших объемах данных (много точек измерения) строка может стать очень длинной, что замедлит работу `Clipboard`.
