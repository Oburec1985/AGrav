# Протокол обмена данными через буфер обмена (Clipboard)

Данный документ описывает программный протокол обмена данными между **Имитатором управления ЦАП** (`Priem.exe`) и **Пультом управления** (`Projzad_zad.exe`).

## 1. Общие принципы
Обмен данными реализован через системный буфер обмена Windows в текстовом (ANSI) формате.
*   **Среда передачи**: Windows Clipboard.
*   **Роли**:
    *   **Пульт**: Инициатор команд. Записывает пакет в буфер, затем ожидает ответа.
    *   **Имитатор**: Исполнитель. Опрашивает буфер по таймеру (интервал ~170 мс), выполняет команду и записывает результат обратно в буфер.

## 2. Формат пакета команды (Пульт -> Имитатор)
Команда представляет собой строку с параметрами, разделенными запятыми. Пакет завершается специальным маркером.

**Структура**: `ID_Команды,Параметр1,Параметр2,...,\Контрольный_код:`

| ID | Описание команды | Параметры | Пример |
|:---|:---|:---|:---|
| 1 | Установка частоты | `Частота` | `1,1000,\20:` |
| 2 | Настройка амплитуды | `Амплитуда, BPS` | `2,5.0,10,\21:` |
| 3 | Управление (Выкл) | `BPS, 0` | `3,\10,0:` |
| 4 | Старт (Вкл) | `Частота, Амплитуда` | `4,1000,5.0,\1:` |
| 8 | **Запуск измерения** | `Точек, Данных` | `8,1024,1,\27:` |
| 13| Завершение работы | `44` | `13,44,\28:` |

*Примечание: Символ `\` является обязательным разделителем конца основной части данных.*

## 3. Формат ответа (Имитатор -> Пульт)
Имитатор возвращает данные (результаты измерений) в тот же буфер обмена.

**Структура данных**: `Val1,Val2,Val3,Val4,...\Статус:`

Для команды измерения (ID 8) данные передаются парами:
*   `Val1, Val2` — Re(1), Im(1)
*   `Val3, Val4` — Re(2), Im(2)
*   ...и так далее.

**Признак готовности**:
После разделителя `\` имитатор записывает **отрицательное** число (соответствующее коду команды с минусом). 
Например, для команды 8 статус будет `-27:`. 
Пульт в цикле ожидания ищет это отрицательное число, чтобы начать разбор данных.

## 4. Особенности реализации
1.  **Таймауты**: Имитатор использует `Sleep(120)` или `Sleep(2600)` (при измерении) для эмуляции задержек оборудования.
2.  **Очистка**: Буфер часто очищается (`Clipboard.Clear`) перед записью или после прочтения для предотвращения повторной обработки одного и того же пакета.
3.  **Разбор строк**: Используются функции `InStr`, `Mid` и `Val` для выделения числовых значений из текстовой строки.
