# Стандарты разработки и Именование

Основные правила написания кода и организации файлов в проектах.

## Общие принципы
- **Язык**: Русский (инструкции, комментарии).
- **Стиль**: Краткость, четкость, минимум воды.
- **Кодировка**: Всегда используй **UTF-8** для файлов с русским текстом.

## Delphi 2010 Specific
- Проект ведется на Delphi 2010.
- При изменении кода необходимо корректировать соответствующую документацию.
- Перекомпиляцию проекта пользователь вызывает самостоятельно.

## Соглашение об именовании
Используй следующие префиксы для переменных и членов классов:

| Тип | Префикс | Пример |
| :--- | :--- | :--- |
| Локальные переменные | `l` | `lnum`, `lResult` |
| Глобальные переменные | `g_` | `g_DBase`, `g_Config` |
| Поля классов (protected) | `f` | `fProperty`, `fInternalState` |
| Члены классов (private/accessible) | `m_` | `m_Session`, `m_Data` |

> [!TIP]
> Старайтесь использовать существующие функции и не дублировать функционал. Анализируйте влияние изменений на другие модули перед началом работ.
