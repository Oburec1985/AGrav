# Стандарт логирования

Этот стандарт является обязательным для всех технических проектов в рамках AGrav.

## 📋 Основные требования

1.  **Идентификация потока (Thread ID)**:
    - Любое сообщение в логе должно содержать ID потока.
    - *Зачем*: Позволяет эффективно отлаживать конкурентные запросы (например, в FastAPI или многопоточных Delphi-приложениях).

2.  **Трассировка функций (Tracing)**:
    - Необходимо логировать **вход** и **выход** из ключевых функций (бизнес-логика, маппинг данных, доступ к БД).
    - *Пример*: `Entering parse_val_with_unit...` -> `Exiting parse_val_with_unit with result 42.0`.

3.  **Уровни логирования**:
    - `INFO`: Важные этапы жизненного цикла (запуск, окончание импорта).
    - `DEBUG`: Подробная трассировка (после стабилизации функционала может быть скрыта, но не удалена).
    - `ERROR`: Исключения с обязательным дампом стека (Traceback).

4.  **Безопасность (Thread-Safe)**:
    - Логгер должен быть потокобезопасным, чтобы избежать перемешивания строк при одновременной записи из разных потоков.
