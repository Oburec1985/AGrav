# 🌍 Локализация и Интернационализация (Delphi / dxgettext)

Данное руководство описывает полный цикл подготовки английской версии для проектов Delphi (на примере **WPExtPack**) с использованием инструментария `dxgettext`.

---

## 🛠 1. Подготовка и установка

Для работы требуются инструменты GNU Gettext, адаптированные для Delphi.

1.  **Установка**: Запустите инсталлятор: 
    `C:\Oburec\OburecGH\sharedUtils\utils\languages\dxgettext-1.2.2.exe`
2.  **Расположение**: По умолчанию инструменты устанавливаются в `C:\Program Files (x86)\dxgettext`.
3.  **Современные утилиты**: Рекомендуется использовать обновленные версии `msgfmt.exe` и `msgcat.exe` (для избежания зависаний), которые находятся здесь:
    `c:\Oburec\OburecGH\sharedUtils\utils\locTools\`
4.  **Настройка окружения**: Добавьте путь `C:\Program Files (x86)\dxgettext` в системную переменную `PATH`.

---

## 📂 2. Извлечение строк (Extraction)

Процесс сбора всех текстовых строк из исходного кода (`.pas`) и ресурсов форм (`.dfm`).

### Командная строка (CLI)
```bash
# Извлечь все строки рекурсивно из текущей папки
dxgettext.exe --delphi -b . -o locale\en\LC_MESSAGES\ --nonascii
```

> [!CAUTION]
> **Обязательный флаг `--nonascii`**: Без этого флага все строки на кириллице (русский язык) будут проигнорированы парсером.

---

## ✍️ 3. Перевод и редактирование

Шаблон `default.po` содержит пары `msgid` (оригинал) и `msgstr` (перевод). Используйте **PoEdit** или VS Code.

> [!IMPORTANT]
> **Кодировка**: Сохраняйте файл строго в **UTF-8 with BOM**. Без этого кириллица в Delphi 2010 может превратиться в некорректные символы.

---

## ⚙️ 4. Сборка и консолидация

### Компиляция ресурсов (.po -> .mo)
```powershell
"c:\Oburec\OburecGH\sharedUtils\utils\locTools\msgfmt.exe" -o locale\en\LC_MESSAGES\default.mo locale\en\LC_MESSAGES\default.po
```

### Объединение файлов (msgcat)
Если сбор проводился из разных источников (разные папки или компоненты), используйте `msgcat` для слияния:
```powershell
msgcat.exe --use-first main.po extra.po -o default.po
```

## 💻 5. Интеграция в код

### В главном файле проекта (.dpr)
```delphi
uses gnugettext;

begin
  // Указываем имя файла .mo (без расширения)
  textdomain('default'); 
  // Включаем поддержку домена для ResourceStrings
  AddDomainForResourceString('default'); 
  // Выбираем язык (имя папки в locale\)
  UseLanguage('en');
  
  Application.Initialize;
  ...
end.
```

### Автоматический перевод форм (Best Practice)
Чтобы не вызывать перевод вручную в каждой форме, создайте базовый класс:

```delphi
type
  TBaseTranslatedForm = class(TForm)
    procedure FormCreate(Sender: TObject);
  end;

procedure TBaseTranslatedForm.FormCreate(Sender: TObject);
begin
  TranslateComponent(Self);
end;
```
*Наследуйте все новые формы от `TBaseTranslatedForm`.*
