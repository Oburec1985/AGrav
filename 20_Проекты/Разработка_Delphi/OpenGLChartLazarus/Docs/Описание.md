# OpenGLChartLazarus: описание

## Цель

Создать кроссплатформенный компонент графиков для Lazarus/FPC с OpenGL-отрисовкой, пригодный для Windows и Linux, а позднее при необходимости для macOS.

Проект должен сохранить полезный опыт Delphi-компонента `cChart`, но не должен следовать его архитектурному шаблону. Старый компонент писался давно и рос стихийно, поэтому новая версия проектируется системно: с разделением модели, рендера, интерактивных инструментов, сериализации и LCL-интеграции.

Код проекта будем вести в:

`C:\Oburec\Projects\Lazarus`

## Источник опыта

Исходный компонент находится в:

`C:\Oburec\OburecGH\sharedUtils\компоненты\chart_dpk\chart`

Главный класс:

- `uChart.pas`
- `cChart = class(TPanel)`

По первичному осмотру `cChart` объединяет несколько ролей:

- визуальный контейнер;
- владелец OpenGL-контекста;
- диспетчер отрисовки;
- менеджер страниц и вкладок;
- владелец объектов графика;
- источник событий мыши, клавиатуры, курсоров и редактирования;
- интеграция с деревом конфигурации и легендой.

Для Lazarus это надо разделить на более переносимые слои.

Delphi-версия не является эталоном наследования. Она нужна, чтобы не потерять рабочие сценарии: страницы, оси, серии, курсоры, легенду, редактирование и сохранение конфигурации.

## Границы первой версии документации

В этой фазе фиксируем:

- целевую классовую структуру новой реализации;

Код, пакет Lazarus, демо-форма и тесты появятся позже отдельным этапом.

## Предварительные архитектурные принципы

- UI-слой должен быть LCL-совместимым.
- OpenGL-контекст должен создаваться через Lazarus-переносимые средства, а не через прямой WinAPI.
- Модель графика должна быть отделена от рендера.
- Интерактивные инструменты должны быть отдельными обработчиками, а не скрытой логикой внутри главного контрола.
- Сохранение настроек должно иметь платформенно-нейтральный формат.
- Каждая новая сущность должна иметь одну главную ответственность и отдельную страницу описания.
