# TChartLayoutRect

## Ответственность

Описание размещения объекта одновременно в относительных и пиксельных координатах.

## Зачем два представления

Float-координаты нужны как основной способ восстановления компоновки при загрузке. В этот момент OpenGL-контекст может еще не быть создан, а фактический размер окна неизвестен.

Пиксельные координаты нужны для реального рисования, hit-test и взаимодействия мышью.

## Поля

- `FloatRect` - относительный прямоугольник внутри родителя.
- `PixelRect` - рассчитанный прямоугольник в пикселях.
- `MinPixelSize` - минимальный допустимый размер.
- `AnchorMode` - правила привязки.
- `SnapMode` - правила примагничивания к соседям.

## Правила

- При загрузке приоритетен `FloatRect`.
- При изменении размера родителя пересчитывается `PixelRect`.
- Если страница слишком мала, `FloatRect` сохраняет намерение компоновки, а `PixelRect` может быть временно ограничен минимальным размером.
- Ручное изменение мышью может обновлять оба представления: пиксельное немедленно, float после нормализации относительно родителя.
