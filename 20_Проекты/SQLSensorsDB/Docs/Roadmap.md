# План развития: SQLSensorsDB (Roadmap)

Документ отслеживает эволюцию проекта от прототипа до промышленного инструмента подбора оборудования.

---

## ✅ Завершено (Milestones)

### Спринт 1: Фундамент
- [x] Проектирование схемы БД (EAV модель).
- [x] Разработка парсера Obsidian (начальная версия).
- [x] Базовый API на FastAPI.
- [x] Прототип UI (React).

### Спринт 2: Портативность и Стабильность (Текущий этап)
- [x] **Streamlined Import**: Парсер стал самодостаточным (авто-создание категорий и маппингов).
- [x] **Robust ETL**: Обработка каскадной кодировки (UTF-8/CP1251) и защита габаритов от ошибок Regex.
- [x] **UI Polish**: Переход к интерфейсу в стиле Яндекс.Маркет (фильтры, динамические колонки).
- [x] **Unit Toggle**: Переключатель G / М/С² с пересчетом данных на лету.

---

## 🚀 В планах (Future Goals)

### Спринт 3: Интеллектуальный подбор (Smart Matching)
- [ ] **Совместимость оборудования**: Логика «Логгер <-> Датчик». Система должна понимать, можно ли подключить выбранный датчик к конкретному входу измерительного модуля (напр. проверка питания IEPE).
- [ ] **Экспорт ТКП**: Генерация коммерческого предложения или спецификации в PDF в один клик.

### Спринт 4: Экосистема
- [ ] **Web-Dashboard**: Расширенная статистика по базе (кол-во в реестре, распределение по брендам).
- [ ] **API для внешних систем**: Доступ к базе датчиков для других внутренних утилит.
