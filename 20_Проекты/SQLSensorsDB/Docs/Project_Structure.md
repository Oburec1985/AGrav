# Структура проекта SQLSensorsDB

В этом документе описано назначение ключевых файлов проекта для понимания архитектуры и быстрого внесения правок.

---

## 🏗 Общая концепция
Проект разделен на две независимые части: **Backend** (обработка данных и API) и **Frontend** (визуализация и поиск). Сборка фронтенда осуществляется инструментом **Vite**.

---

## 📂 Backend (Python, FastAPI, SQLModel)
*Расположение: `src/backend/`*

### Основные скрипты:
- **`import_obsidian.py`**: ETL-процесс. Читает Markdown из Obsidian, нормализует параметры, настраивает связи и наполняет базу. Теперь является самодостаточным (автоматически создает категории и производителей).
- **`seed.py`**: Скрипт для начальной тестовой забивки базы. Используется в основном для отладки, не обязателен для работы с реальными данными Obsidian.

### Директория `app/`:
- **`main.py`**: Точка входа в API. Содержит маршруты (`/products`, `/categories`, `/manufacturers`), логику комбинированного поиска и фильтрации.
- **`models.py`**: Описание структур таблиц базы данных с использованием SQLModel (Pydantic + SQLAlchemy).
- **`database.py`**: Настройка подключения к SQLite (`sensors.db`) и управление сессиями.
- **`logger.py`**: Конфигурация логирования запросов и ошибок.

---

## 📂 Frontend (React, Vite, TypeScript)
*Расположение: `src/frontend/`*

### Сборка и конфигурация:
- **`vite.config.ts`**: Настройка инструмента Vite. Определяет правила сборки, порты и алиасы путей.
- **`package.json`**: Список зависимостей (React, Lucide icons, Axios) и команды запуска (`npm run dev`).

### Директория `src/`:
- **`App.tsx`**: Корневой компонент. Настраивает маршрутизацию (Routing) между основной таблицей и карточкой продукта.
- **`components/Dashboard.tsx`**: **Центральный узел UI**. Содержит всю логику фильтрации, поисковую строку, боковую панель и главную таблицу результатов.
- **`components/ProductPage.tsx`**: Страница детального просмотра. Форматирует характеристики, выводит изображения и отвечает за генерацию PDF/печать.
- **`index.css`**: Глобальные стили проекта. Здесь реализована темная тема, glassmorphism-эффекты и Tailwind-подобные утилиты.
- **`constants.ts`**: Глобальные константы (коэффициенты пересчета юнитов, названия параметров).

---

## 🛠 Инструменты
- **Vite**: Сборщик фронтенда. Обеспечивает мгновенное обновление интерфейса при изменении кода (HMR).
- **SQLite**: База данных в виде одного файла (`sensors.db`).
- **FastAPI**: Высокопроизводительный фреймворк для API.
