# ALF (AGrav Lost & Found)

Некоммерческий общественный сервис для поиска потерянных вещей, животных и людей.

## 🎯 Цель проекта
Создать удобную систему учета и поиска "потеряшек". Проект разрабатывается как общественная инициатива ("для себя и для людей") без рекламы и платных функций.

## 📢 Настройка уведомлений (Telegram и Почта)
Чтобы владельцы объявлений получали уведомления об откликах, необходимо настроить переменные окружения в файле `docker-compose.yml`.

### 1. Настройка Telegram (Мгновенные уведомления)
1. **Создайте бота**: Напишите [@BotFather](https://t.me/BotFather) команду `/newbot`, выберите имя и получите **API Token**.
2. **Пропишите токен**: Откройте `docker-compose.yml`, найдите параметр `TELEGRAM_BOT_TOKEN` и вставьте туда свой токен.
3. **Запустите бота**: Найдите своего бота в поиске Telegram и нажмите **Start**.
4. **Узнайте свой ID**: Нажмите кнопку "Узнать свой ID" прямо на сайте при создании объявления (она направит вас в бот).

### 2. Настройка Почты (Email уведомления)
Для отправки писем рекомендуется использовать "Пароль приложения" (это безопаснее основного пароля).
1. **Получите пароль приложения**:
   - [Инструкция для Gmail](https://support.google.com/accounts/answer/185833)
   - [Инструкция для Яндекс.Почты](https://yandex.ru/support/id/authorization/app-passwords.html)
   - [Инструкция для Mail.ru](https://help.mail.ru/mail/security/packer/pass)
2. **Заполните данные в `docker-compose.yml`**:
   - `SMTP_HOST`: например, `smtp.gmail.com` или `smtp.yandex.ru`.
   - `SMTP_PORT`: обычно `587` (TLS) или `465` (SSL).
   - `SMTP_USER`: ваша почта (например, `my-service@gmail.com`).
   - `SMTP_PASSWORD`: тот самый **пароль приложения**, который вы получили в п.1.

## 🚀 Быстрый запуск (One-Click Start)
Мы придерживаемся принципа **"Запуск одной кнопкой"**.
1. **Запустите файл `run_alf.bat`** в корне проекта.
2. Скрипт сам поднимет базу данных, бэкенд, фронтенд и бота.


Мы используем технологию **Docker**, чтобы тебе не пришлось устанавливать базу данных, Python и кучу библиотек вручную. Docker — это как "контейнер", в котором уже все настроено и готово к работе.

### Что такое `docker-compose.yml`?
Это "карта-инструкция" для Docker. В ней написано: "Возьми базу данных PostgreSQL, добавь туда настройки карт PostGIS, запусти сервер на Python и соедини их между собой".

### Как запустить за 3 шага:

1. **Установи Docker Desktop** (если еще нет) с официального сайта [docker.com](https://www.docker.com/).
2. **Открой терминал** (или командную строку) в папке проекта.
3. **Введи одну команду**:
   ```bash
   docker compose up -d
   ```

**Готово!** Сайт будет доступен по адресу `http://localhost:8000`. 
*(Примечание: на этапе разработки там может быть просто страница с API или заглушка).*

## 🛠 Команды управления
- `docker compose stop` — поставить проект на паузу.
- `docker compose start` — продолжить работу.
- `docker compose logs -f` — посмотреть, что происходит "под капотом" (логи).
