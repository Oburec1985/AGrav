# Схема базы данных: ALF

Для реализации проекта используется **PostgreSQL** с расширением **PostGIS**.

## 1. Таблица: `items` (Объявления)
Основная таблица с информацией о потерях и находках. Использует **PostGIS** для гео-поиска.

| Поле | Тип | Описание |
| :--- | :--- | :--- |
| `id` | Integer (PK) | Уникальный ID. |
| `title` | String | Заголовок объявления. |
| `description` | Text | Подробное описание. |
| `category` | String | Категория (например, `cat_pets_dog`). |
| `type` | Enum | `LOST` или `FOUND`. |
| `status` | Enum | `ACTIVE`, `RETURNED`, `ARCHIVED`. |
| `location` | **Geometry(POINT, 4326)** | Географическая точка (WGS84). |
| `location_str` | String | Читаемый адрес. |
| `image_url` | String | Ссылка на основное фото. |
| `reward_info` | String | Информация о вознаграждении. |
| `is_contact_public` | Boolean | Видны ли контакты гостям. |
| `contact_phone` | String | Телефон для связи. |
| `contact_email` | String | Email для связи. |
| `edit_password` | String | Пароль для редактирования. |
| `event_date` | Timestamp | Дата происшествия. |
| `created_at` | Timestamp | Дата создания записи. |

## 2. Таблица: `messages` (Отклики)
Сообщения от пользователей, нашедших или видевших вещь.

| Поле | Тип | Описание |
| :--- | :--- | :--- |
| `id` | Integer (PK) | ID сообщения. |
| `item_id` | Integer (FK) | Ссылка на объявление. |
| `content` | Text | Текст сообщения. |
| `contact_info` | String | Контакты отправителя. |
| `photos_urls` | JSON | Массив ссылок на фото-доказательства. |
| `created_at` | Timestamp | Дата отправки. |

---

### Примечания к реализации:
1. **Гео-запросы**: Поиск в радиусе осуществляется через функцию `ST_DWithin` и индекс `GIST` на поле `location`.
2. **Иерархия категорий**: Категории реализованы как строки с префиксами (например, `cat_pets_...`), что позволяет искать как по конкретному типу, так и по всей группе.
3. **Безопасность**: Управление объявлением осуществляется через `edit_password`, пока не внедрена полноценная авторизация через Telegram.
