# Описание API: ALF

API строится по принципу REST и использует JSON для обмена данными. Все запросы, требующие авторизации, должны содержать `Bearer token`.

## 1. Публичные эндпоинты (Доступны без регистрации)

### `GET /api/v1/items`
Поиск и фильтрация объявлений.
- **Параметры**:
  - `lat`, `lon`: Координаты центра поиска.
  - `radius`: Радиус в метрах (по умолчанию 5000).
  - `type`: `LOST` или `FOUND`.
  - `category_id`: ID категории.
  - `query`: Поиск по тексту (заголовок/описание).
  - `offset`, `limit`: Пагинация.

### `GET /api/v1/items/{id}`
Детальная информация.

### `GET /api/v1/stats`
Глобальная статистика системы.
- **Ответ**:
  - `total_lost`: Всего потеряно.
  - `total_found`: Всего найдено (объявлений).
  - `total_returned`: Успешно возвращено владельцам.
  - `active_search`: В активном поиске сейчас.

### `GET /api/v1/categories`
Получение списка всех доступных категорий.

---

## 2. Авторизация

### `POST /api/v1/auth/telegram`
Вход через Telegram. Принимает данные от виджета или бота.
- **Body**: `hash`, `auth_date`, `id`, `first_name`, и т.д.
- **Ответ**: `access_token`.

---

## 3. Личный кабинет и Объявления (Требуется Auth)

### `POST /api/v1/items`
Создание нового объявления.
- **Body**: `title`, `description`, `category_id`, `type`, `lat`, `lon`, `is_contact_public`, `reward_info`.

### `PATCH /api/v1/items/{id}`
Редактирование или смена статуса (например, закрытие объявления).

### `GET /api/v1/me`
Получение профиля текущего пользователя (карма, свои объявления).

---

## 4. Медиа

### `POST /api/v1/media/upload`
Загрузка изображений.
- **Multipart Form**: `file`.
- **Ответ**: `url` загруженного файла.

---

## 5. Коммуникация (Требуется Auth)

### `GET /api/v1/chats`
Список активных чатов пользователя.

### `GET /api/v1/chats/{chat_id}/messages`
История сообщений в чате.

### `POST /api/v1/chats/{chat_id}/messages`
Отправка сообщения.

---

## 💡 Особенности реализации:
1. **Rate Limiting**: Ограничение количества запросов на создание объявлений для защиты от спама.
2. **Geo-Indexing**: Поиск по координатам (`/items`) выполняется на уровне БД через PostGIS для максимальной скорости.
3. **WebSockets**: (Опционально) Для мгновенного получения сообщений в чате.
