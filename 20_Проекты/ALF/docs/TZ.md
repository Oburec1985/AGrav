# Техническое задание: ALF (AGrav Lost & Found)

## 1. Общие сведения
ALF — это некоммерческий общественный сервис (Web + Android) для регистрации фактов потери или находки объектов. Основная философия: "Для себя и для людей" — отсутствие рекламы, платных функций и подписок.

## 2. Основной функционал
### 2.1. Регистрация объявлений
- **Типы**: Потеряно / Найдено.
- **Категории**: Животные, Документы, Техника, Ключи, Прочее.
- **Поля**: 
  - Заголовок (кратко).
  - Подробное описание.
  - Дата и время события.
  - Местоположение (координаты + адрес).
  - Фотографии (до 5 шт).
  - Вознаграждение (да/нет/сумма).
  - Контактные данные (скрыты до подтверждения интереса или открыты).

### 2.2. Поиск и Фильтрация
- По радиусу от точки на карте.
- По ключевым словам и тегам.
- По дате (период).
- По категориям.

### 2.3. Интеллектуальное сопоставление (Matching)
- Система в фоновом режиме сравнивает новые объявления "Нашел" с базой "Потерял".
- Критерии: Пересечение гео-зон, совпадение категорий, схожесть описания (ключевые слова).
- Уведомление пользователя о возможных совпадениях.

### 2.4. Роли пользователей
- **Гость**: Просмотр объявлений, поиск.
- **Зарегистрированный пользователь**: Создание объявлений, чат, управление своими записями.
- **Модератор**: Удаление спама, коррекция категорий.

## 3. Технические требования (Open Source Stack)
- **Бэкенд**: Python (FastAPI).
- **БД**: PostgreSQL + PostGIS (Полностью бесплатное и мощное решение для гео-данных).
- **Картография**: OpenStreetMap (Leaflet / MapLibre GL) — бесплатная альтернатива Google Maps.
- **Фронтенд (Web)**: Next.js.
- **Мобильное приложение**: Android (Kotlin).
- **Хранение медиа**: Локальное хранилище сервера или бесплатные лимиты S3-совместимых сервисов.

## 4. Безопасность и Доверие
- Верификация через Telegram-бота (бесплатно и удобно).
- Система "Кармы" или доверия пользователей на основе успешных возвратов.
- Полная прозрачность данных для сообщества.
