# Технический отчет — 03 мая 2026

## ✅ Выполненные задачи

### 1. Автоматизация инфраструктуры (One-Click Start)
Реализован принцип бесшовного запуска проекта на локальной машине разработчика.
- **Создан файл `run_alf.bat`**:
  - **Автозапуск Docker**: Скрипт проверяет статус Docker Desktop и автоматически запускает его, если он выключен.
  - **Умное ожидание**: Внедрен цикл проверки готовности Docker API перед запуском контейнеров.
  - **Host-side Environment**: Автоматизировано создание Python `venv` и установка зависимостей (`requirements.txt`) для запуска бэкенда напрямую в ОС (без Docker-контейнера для кода), что ускоряет разработку.
  - **Frontend Fix**: Исправлена ошибка Turbopack в Next.js на Windows путем принудительного переключения на Webpack в режиме разработки.
  - **Авто-браузер**: Скрипт автоматически открывает `http://localhost:3000` после успешного старта всех систем.

### 2. Обновление дизайна (UI/UX)
- **Рефакторинг панели фильтров**:
  - Убран эффект "мутного стекла" (glassmorphism) с левой панели, так как он мешал читаемости текста на фоне пестрой карты.
  - Внедрен новый фон: **Slate 800 (rgba(30, 41, 59, 0.92))**. Это контрастный серо-голубой цвет, который обеспечивает отличную читаемость и выглядит премиально.
  - Скорректированы границы и тени для лучшего визуального отделения панели от карты.
  - Оптимизирована работа со шрифтами и отступами в боковом меню.

## 📜 Новое системное правило
**"Запуск одной кнопкой"**: Любое изменение в коде или базе данных должно учитывать, что проект обязан запускаться через `run_alf.bat`. Ручные манипуляции с консолью для старта базы или серверов должны быть сведены к нулю.
