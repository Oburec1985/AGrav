- **ALF / Security & Authentication**: Реализован полный цикл авторизации и защиты данных.

## 2026-04-27
- **Python_Excel / полный цикл обработки**: добавлен `run_salary_pipeline.bat`, который запускает парсинг зарплатных картинок, предлагает выбрать Excel-файл по подстроке имени и затем переносит суммы из `result.csv` в Excel. `update_excel_template.py` теперь принимает не только точный путь к `.xlsx`, но и частичное совпадение имени файла через `resolve_excel_file`.
- **Python_Excel / устойчивость батника**: выбор Excel вынесен из сложной inline PowerShell-команды в `select_excel_file.ps1`, а `run_salary_pipeline.bat` переведен в ASCII/CRLF-формат, чтобы `cmd.exe` не ломал команды на русских символах и управляющих знаках. Батник теперь автоматически выбирает первый Excel по подстроке `РСС` и делает `pause` в конце/при ошибке, чтобы окно не закрывалось без диагностики.
- **Python_Excel / кодировка путей Windows**: `select_excel_file.ps1` передает выбранный Excel обратно в `.bat` коротким DOS-путем, чтобы русские символы в каталоге `00_Система\Инструменты` не превращались в `????` при чтении временного файла из `cmd.exe`.
- **Python_Excel / короткие пути .xlsx**: `update_excel_template.py` разворачивает 8.3 Windows-пути через `GetLongPathNameW`, чтобы короткое имя `.XLS` для реального `.xlsx` не приводило к ошибке `openpyxl does not support the old .xls file format`.
- **Python_Excel / экономия времени**: в `run_salary_pipeline.bat` добавлен вопрос о необходимости заново готовить `result.csv`; по умолчанию OCR-парсинг пропускается и используется уже готовый результат.
- **Python_Excel / запись Excel**: `run_salary_pipeline.bat` обновляет выбранный Excel на месте через `--output` с тем же путем. Лишний вопрос о перезаписи убран; параметры листа и колонок остаются с дефолтами, которые можно принять Enter или переопределить вручную.
- **Python_Excel / единицы измерения**: в `update_excel_template.py` добавлен параметр `--value-divisor`; батник передает `1000`, чтобы суммы из `result.csv` записывались в Excel в тысячах.
- **Python_Excel / OCR ФИО**: `parse_zp_image.py` больше не использует позицию блока как главный источник ФИО. Распознавание имени переведено на левую ячейку заголовка без месяца, добавлены OCR-варианты `otsu/adaptive/sharp/gray`, масштабирование по высоте строки, русский whitelist и сравнение со словарем с отдельным учетом фамилии и хвоста инициалов. Это должно лучше различать `Горенков А.В.`, `Скрипник А-й.А.` и `Скрипник А-р А.`.
- **Python_Excel / parse_zp_image.py**: ускорен разбор зарплатных JPG. Скрипт теперь использует найденные горизонтальные и вертикальные линии таблицы, быстро определяет общую строку `ИТОГО к выплате` по нижним строкам первых блоков и распознает только ячейки итоговой выплаты во второй колонке каждого блока. Добавлены срез рамок ячейки перед OCR, числовой OCR-конфиг для денежных значений и позиционный fallback для одинаковой структуры блоков.
- **Python_Excel / диагностика окружения**: в `parse_zp_image.py` добавлен лог Python executable, версии Python, OpenCV, NumPy, Pillow, pytesseract и Tesseract, чтобы фиксировать рабочее окружение запуска.
- **Python_Excel / диагностика геометрии OCR**: добавлены подробные `[GEOM]`, `[SCAN]`, `[CELL]` и `[FALLBACK]` логи для сверки координат OpenCV-линий, LLM-визуальной гипотезы строки `ИТОГО к выплате`, OCR меток и распознавания сумм.
- **Python_Excel / логирование**: `ocr.log` теперь очищается в начале каждого запуска `parse_zp_image.py`, чтобы в диагностике оставались только данные текущего прогона.
- **Python_Excel / геометрия блоков**: по диагностическому логу исправлена группировка вертикальных линий. Вместо скользящих пар колонок используется группировка независимых блоков по 3 вертикальные линии; отключен опасный fallback выбора максимального числа из нижних строк.
- **Python_Excel / качество OCR**: добавлен отдельный OCR-режим для денежных ячеек с несколькими вариантами предобработки и фильтром правдоподобности итоговой выплаты. Имена в CSV теперь берутся из `employees.txt` через сопоставление OCR-заголовка со словарем по коэффициенту похожести; позиция блока используется только как fallback.
- **Python_Excel / OCR сумм**: исправлен язык OCR денежных ячеек с `eng` на `rus`, так как локальный `tessdata` содержит русский пакет; это устраняет ошибки `eng.traineddata` и пустые `amount_candidates`.
- **Python_Excel / стабилизация инструмента**: удален неиспользуемый экспериментальный код и временные подробные OCR-логи, добавлены пользовательская инструкция [Python_Excel_ПарсерЗП](Python_Excel_ПарсерЗП.md) и техническое описание [parse_zp_image_tech](../00_Система/Инструменты/Python_Excel/parse_zp_image_tech.md).
- **Python_Excel / Excel update**: `update_excel_template.py` теперь загружает значения из `result.csv`/`results.csv` и обновляет Excel-шаблон через CLI-параметры листа, колонок и стартовой строки.
