# Заметки: parse_zp_image.py

## Задача

Экспорт результатов парсинга зарплатных изображений в CSV для дальнейшей обработки в Excel.

## Текущее Решение

- Целевая строка: `ИТОГО к выплате`.
- Строка `СУММА` не используется как итоговая выплата.
- OpenCV ищет горизонтальные и вертикальные линии таблицы.
- Вертикальные линии группируются в независимые зарплатные блоки по 3 линии: левая граница, разделитель, правая граница.
- Для январских JPG строка `ИТОГО к выплате` находится на `row_idx = 20`.
- Скрипт распознает только нужную денежную ячейку каждого блока, а не всю таблицу.

## Ключевые Исправления 2026-04-27

- Исправлена ошибка скользящей группировки колонок: шаг 2 создавал псевдоблоки и узкие ячейки суммы.
- Добавлена `build_salary_blocks(x_coords)` с группировкой по 3 вертикальные линии.
- Удален опасный fallback выбора максимального числа из нижних строк.
- Добавлен `ocr_amount_cell`: масштаб x6, белая рамка, несколько режимов бинаризации и `psm`.
- OCR сумм использует `lang='rus'` с цифровым whitelist, так как локально есть `rus.traineddata`.
- Сумма принимается только в диапазоне правдоподобной итоговой выплаты `50000..500000`.
- `employees.txt` используется как словарь эталонных ФИО: OCR имени сопоставляется со словарем, а в CSV пишется только словарное имя.
- `ocr.log` очищается при каждом запуске.
- Временные подробные диагностические логи `[GEOM]`, `[SCAN]`, `[CELL]`, `[FALLBACK]` убраны из боевого режима.

## 2026-04-27: связка с Excel

- `update_excel_template.py` теперь сам загружает данные из `result.csv` или `results.csv`.
- Формат CSV: `ФИО;Значение`.
- Скрипт запускается из командной строки и принимает Excel-файл, имя листа, колонку ФИО, целевую колонку и стартовую строку.
- Колонки можно задавать буквами Excel или 1-based номерами.
- Исходный Excel не перезаписывается: результат сохраняется в `*_updated.xlsx`, если не задан `--output`.
