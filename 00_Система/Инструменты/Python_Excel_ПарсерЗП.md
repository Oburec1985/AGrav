# Python_Excel

Системный инструмент для пакетной обработки Excel и связанных табличных данных через Python. Основной текущий сценарий: распознавание зарплатных ведомостей из JPG и подготовка `result.csv` для дальнейшей загрузки в Excel.

## Установка

Нужен Python с библиотеками:

```powershell
pip install pytesseract Pillow opencv-python numpy openpyxl pandas
winget install UB-Mannheim.TesseractOCR
```
https://digi.bib.uni-mannheim.de/tesseract/tesseract-ocr-w64-setup-5.4.0.20240606.exe

Нужен установленный Tesseract OCR. Скрипт ищет его в стандартных путях Windows:

- `C:\Program Files\Tesseract-OCR\tesseract.exe`
- `C:\Program Files (x86)\Tesseract-OCR\tesseract.exe`

Русский языковой пакет должен быть доступен как `rus.traineddata`. Если его нет в системном Tesseract, `parse_zp_image.py` использует локальную папку:

```text
00_Система\Инструменты\Python_Excel\tessdata
```

## Парсинг Зарплатных JPG

Файлы:

- `parse_zp_image.py` — парсер зарплатных изображений.
- `employees.txt` — эталонный список сотрудников, по одному ФИО в строке.
- `result.csv` — результат распознавания.
- `update_excel_template.py` — перенос значений из `result.csv` в Excel-шаблон.
- `run_salary_pipeline.bat` — запуск полного цикла: парсинг картинок, выбор Excel и запись сумм.
- `select_excel_file.ps1` — вспомогательный выбор Excel-файла по подстроке для батника.
- `ocr.log` — лог текущего запуска, очищается при каждом запуске.

## Быстрый Запуск Всего Цикла

Из папки инструмента запустить:

```cmd
run_salary_pipeline.bat
```

Батник делает полный рабочий проход:

1. Запускает `parse_zp_image.py` и формирует свежий `result.csv`.
2. Если на вопрос `Run image parsing and rebuild result.csv? [y/N]` ответить `N` или просто нажать Enter, OCR пропускается и используется уже готовый `result.csv`/`results.csv`.
3. Сам ищет Excel-файл по подстроке `РСС` и подставляет первый найденный путь.
4. Запрашивает лист, колонку ФИО, колонку суммы и стартовую строку. Enter оставляет дефолт: лист `Итог`, ФИО `B`, сумма `H`, стартовая строка `1`.
5. Запускает `update_excel_template.py` и обновляет выбранный Excel на месте.

Запускать из папки инструмента:

```powershell
cd E:\Oburec\AGrav\00_Система\Инструменты\Python_Excel
.\parse_zp_image.py
```

Скрипт обрабатывает все `*.jpg`, `*.jpeg`, `*.png` в текущей папке, распознает строку `ИТОГО к выплате` в каждом зарплатном блоке и пишет CSV:

```csv
ФИО;Значение
```

## Перенос Result.csv В Excel

После распознавания можно заполнить Excel-шаблон:

```powershell
.\update_excel_template.py "РСС" --sheet "Итог" --name-col B --target-col H --start-row 16
```

Параметры:

- `excel_file` — путь к исходному `.xlsx` или подстрока имени файла, например `РСС`.
- `--sheet` — имя листа.
- `--name-col` — колонка с ФИО в Excel, можно буквой (`B`) или номером (`2`).
- `--target-col` — колонка, куда записывать суммы.
- `--start-row` — первая строка с сотрудниками.
- `--csv` — путь к CSV; если не задан, ищется `result.csv`, затем `results.csv`.
- `--search-dir` — папка для поиска Excel по подстроке; по умолчанию текущая папка.
- `--value-divisor` — делитель суммы перед записью в Excel. Батник использует `1000`, чтобы записывать суммы в тысячах.
- `--output` — путь для сохранения; если не задан, создается файл `*_updated.xlsx`. Батник передает сюда выбранный Excel-файл, чтобы обновить его на месте.

Скрипт читает `ФИО;Значение` из CSV, сопоставляет ФИО с Excel и записывает суммы в целевую колонку. При запуске через батник значения делятся на 1000. Исходный Excel обновляется на месте.

## Правила Подготовки

- Перед запуском проверь `employees.txt`: порядок и написание сотрудников должны соответствовать ведомостям.
- `employees.txt` является словарем ФИО для коррекции OCR: в CSV попадает ближайшее эталонное имя из словаря.
- Если в ведомости есть однофамильцы, проверь дефисы и инициалы в `employees.txt`: они используются при сравнении, чтобы не схлопывать разных людей в один ключ.
- Картинки ведомостей должны лежать рядом со скриптом.
- После запуска проверь `ocr.log`: в норме должны быть строки с количеством блоков, выбранной строкой `ИТОГО к выплате` и найденными суммами.
- Если сумма не найдена, скрипт пишет `WARNING` и не подставляет случайное значение.
