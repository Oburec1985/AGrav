# Метод парсинга и структурирования технической литературы (AGrav Standard)

Этот документ описывает стандарт и пошаговый алгоритм Antigravity по превращению "сырых" материалов (PDF, OCR, фото) в премиальную техническую базу знаний в формате Markdown.

## 1. Инструментарий (Tech Stack)

### Базовый (Legacy / Fast)
- **библиотека PyMuPDF (fitz)**: Извлечение "сырого" текста и картинок.
- **Скрипт `00_Система/Инструменты/Парсинг_Книг/parse_pdf.py`**: Быстрый, но "глупый" парсинг без учета структуры.

### Умный (Smart Edition — Рекомендуется)
- **Docling (IBM)**: Глубокое распознавание структуры (таблицы, иерархия) — локально и бесплатно.
- **Gemini 1.5 Flash (Cloud AI)**: Максимальный интеллект (понимание графиков, формул) — облачно и бесплатно (до 15 RPM).
- **Скрипт `00_Система/Инструменты/Парсинг_Книг/smart_parse_pdf.py`**: Ультимативный стандарт AGrav. Умеет автоматически извлекать изображения, строить таблицы, расставлять якоря страниц и ссылки навигации полностью локально.

## 3. Инструкция: Двухшаговый метод (Standard)

Этот метод разделяет парсинг и интеллектуальный перевод для максимальной надежности и контроля качества.

### Шаг 1: Смарт-парсинг (PDF -> Markdown)
Используйте локальный парсер для извлечения структуры и изображений:
```bash
python 00_Система/Инструменты/Парсинг_Книг/smart_parse_pdf.py "путь/к/книге.pdf" "папка/вывода" --mode docling
```
*   **Результат**: MD-файл с якорями страниц и папка `Attachments/` с картинками.

### Шаг 2: Интеллектуальный перевод (MD -> RU)
Используйте скрипт перевода через LLM (Gemini) для адаптации терминологии:
```bash
python 00_Система/Инструменты/Перевод_MD/smart_translate_md.py "папка/вывода/файл.md" "папка/вывода/файл_RU.md" --api-key "ВАШ_КЛЮЧ"
```
*   **Результат**: Качественно переведенная книга на русском языке с сохранением оригинальной верстки и ссылок.

## 4. Подготовка и декомпозиция (Phasing)

1.  **Парсинг структуры**: Первым делом анализируется оглавление оригинала.
2.  **Создание скелета**: Формируется файл с якорями для каждой страницы и раздела для быстрой навигации.
    - Пример: `<a name="p1"></a> ## 🏷️ Название (стр. 1)`
3.  **Визуальные активы**: Все изображения сохраняются в папку `Attachments/` с понятными именами (например, `image_p5_1.jpeg`).

## 5. Стандарт оформления (Style Guide)

Для достижения "Wow-эффекта" и высокой читаемости используются:

-   **Emoji-индикаторы**:
    - 📗 — Заголовок книги
    - 📌 — Оглавление
    - 🏷️ — Страницы/Разделы
    - 📊, 📐, 📉 — Тематические блоки
-   **Callout блоки (GitHub/Obsidian style)**:
    - `> [!IMPORTANT]` — Ключевые термины, определения, законы.
    - `> [!TIP]` — Практические советы, лайфхаки, кейсы.
    - `> [!NOTE]` — Примечания, исторические справки, оригинальное содержание.
    - `> [!CAUTION]` / `> [!WARNING]` — Критические ошибки, ограничения метода.
-   **Навигация**:
    - Ссылка наверх: `[🔝 Сверху](#top)` после каждого заголовка страницы.
    - Интерактивное оглавление в начале документа.

## 6. Техническая очистка контента

1.  **Склеивание строк**: Удаление переносов строк, возникших при OCR, чтобы текст корректно обтекал изображения и был адаптивным.
2.  **Акценты**: Выделение **жирным** ключевых слов, на которых должен фокусироваться взгляд при быстром сканировании.
3.  **Математика**: Оформление формул (по возможности в LaTeX, если поддерживается, иначе — четко отформатированным текстом/изображением).

## 7. Технические сложности и решения

При работе с `Modal Testing` и аналогичными книгами возникали следующие проблемы:

1.  **Артефакты OCR/Форматирования**:
    - *Проблема*: Лишние переносы строк в середине предложений.
    - *Решение*: Применение регулярных выражений (`regex`) в Python для удаления `\n`, если следующая строка начинается со строчной буквы.
2.  **Потеря качества формул**:
    - *Проблема*: Сложные математические зависимости могут "разлетаться" при текстовом экспорте.
    - *Решение*: Вырезание формул как изображений или ручная правка в LaTeX.
3.  **Сдвиг иллюстраций**:
    - *Проблема*: Изображения часто извлекаются пачкой в конце страницы.
    - *Решение*: Ручное перемещение тегов `![Image](...)` в соответствующие места текста по смыслу.
4.  **Кодировка спецсимволов**:
    - *Проблема*: Греческие буквы и спецзнаки могут превратиться в "кракозябры".
    - *Решение*: Использование `utf-8-sig` при сохранении `.md` и проверка шрифтов в PDF.

## 8. Алгоритм выполнения (Execution Pipeline)

1.  **Этап 0: Выбор режима**. 
    - Для простых текстов — `parse_pdf.py`.
    - Для книг с таблицами и сложной версткой — `smart_parse_pdf.py` (режим `docling`).
    - Для "критических" материалов с формулами — `smart_parse_pdf.py` (режим `gemini`).
2.  **Этап 1: Оглавление и Якоря**. Создается полная структура ссылок.
2.  **Этап 2: Итеративный перенос контента**. Текст переносится постранично или поглавно. 
    - *Важно*: Каждая страница в Markdown начинается с ее номера и ссылки на верх.
3.  **Этап 3: Инъекция графики**. Изображения вставляются максимально близко к контексту упоминания.
4.  **Этап 4: Смысловое "подсвечивание"**. ИИ анализирует текст и добавляет блоки `[!TIP]` или `[!IMPORTANT]`, которых не было в оригинале в таком виде, но которые вытекают из смысла.
5.  **Этап 5: Финальная проверка навигации**. Проверка всех ссылок в оглавлении.

