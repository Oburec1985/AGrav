# Инструменты сервера Excel MCP (программа ExcelMcpPrj.exe)

Этот документ предоставляет подробную информацию обо всех доступных инструментах на сервере Excel MCP. В настоящий момент это не классический MCP
Алгоритм работы указан ниже.
Программа написана на Pascal соответственно пути записываются в формате C:\path\... (с обратными слешами)

## Принцип работы
Перед работой следует внимательно изучить все функции для продумывания оптимального алгоритма работы с Excel файлами
Программа работает в режиме сервера, ожидая команд. Взаимодействие происходит через файлы в директории `excel-mcp`:

1.  **Отправка команды**: Чтобы отдать команду, клиент (например, Gemini) создает файл `command.in` в папке `excel-mcp`. Файл должен содержать одну строку с командой (например, `create_workbook`).
2.  **Выполнение команды**: Сервер постоянно отслеживает появление файла `command.in`. Как только файл обнаружен, сервер читает из него команду, выполняет её, а затем удаляет `command.in`.
3.  **Получение результата**: После выполнения команды сервер записывает результат (сообщение об успехе или описание ошибки) в файл `command.out` в той же директории.
4.  **Проверка клиентом**: Клиент может прочитать файл `command.out`, чтобы узнать результат выполнения своей команды.
5.  **Старт работы**:
	5.1 При старте работы LLM сперва должна проверить запущена ли ExcelMcpPrj.exe. Проверить запуск можно по наличию процесса в системе или по лог-файлу `Log.txt`.

	**5.1.1 ВАЖНО: Проблема с запуском LLM Gemini**
	Прямой запуск сервера из окружения Gemini (`run_shell_command`) приводит к нестабильной работе. Процесс запускается, но Gemini не может корректно отследить его состояние, что приводит к "зависанию" модели и последующему аварийному завершению сервера при отмене операции.

	**Надежное решение — запуск через `.bat` файл:**
	- Создайте в папке `excel-mcp` файл `start_server.bat` следующего содержания:
	  ```bat
	  @echo off
	  cd /d "%~dp0%"
	  START "MCP Server" ExcelMcpPrj.exe
	  ```
	- Запуск этого `.bat` файла создает полностью независимый процесс сервера в отдельном консольном окне. Gemini может запустить этот файл и, не зависая, продолжить работу, взаимодействуя с уже запущенным сервером.

	5.2 Если сервер не запущен, его следует запустить с помощью `start_server.bat`
	5.3 Далее LLM добавляет команды в файл `command.in` и проверяет их выполнение считывая `command.out`. Сервер после обработки command.in удаляет файл, таким образом LLM понимает, что команда обработана.
	   5.4 Сервер может обрабатывать команды с задержкой. Вместо фиксированного ожидания, рекомендуется использовать следующий алгоритм:
    а. После отправки команды в `command.in`, начать проверку наличия `command.out` с коротким интервалом (например, 1 секунда).
    б. Если файл `command.out` не появляется, продолжать проверку, увеличивая общее время ожидания до 15 секунд.
    в. Если по истечении 15 секунд файл так и не появился, следует считать, что сервер не отвечает, и принять решение о дальнейших действиях (например, уведомить пользователя или попытаться перезапустить сервер).
	5.5 При открытии или создании книги следует уточнять имена листов, т.к. листы могут называться по разному в разных регионах, или может переименовать пользователь. Если создаешь новую книгу можно обращаться к листу не по имени а по индексу
	5.6 При чтении информации в книге следует пользоваться командой get_sheet_dimensions чтобы знать какие ячейки читать командой read_range
6. Если в течении 15 секунд LLM не получила ответ в commnd.out следует прекратить попытки;
7. **Завершение работы**: Команда `quit_excel` закрывает только приложение Excel, но **не останавливает сервер** (`ExcelMcpPrj.exe`). Для полной остановки сервера необходимо завершить его процесс. Для этого можно использовать команду `stop_server`.

Этот файловый механизм обеспечивает простое асинхронное взаимодействие с запущенным сервером.

**обработка нескольких команд**
допустимо записать сразу несколько команд в файл cvommand.in, на каждую команду будет сформирована своя строка в файле command.out
Пример:
 set_cell --sheet_name "Лист 1" --cell "B3" --value "str1"
 set_cell --sheet_name "Лист 1" --cell "B4" --value "str2"
 set_cell --sheet_name "Лист 1" --cell "B5" --value "str3" 

### Выполнение команд через HTTP

В дополнение к файловому обмену, сервер `ExcelMcpPrj.exe` позволяет выполнять команды напрямую через HTTP GET-запросы. Сервер принимает запросы на порту `8080`. Этот способ является предпочтительным для интеграций.

**Формат запроса:**
`http://localhost:8080/?command=КОМАНДА`

Где `КОМАНДА` - это полная строка команды, идентичная той, что используется в файле `command.in`.

**Важно:** Все специальные символы в команде (пробелы, кавычки, обратные слеши и т.д.) должны быть корректно закодированы в URL (URL-encoded).

**Примеры:**

*   **Проверить статус сервера (без параметра `command`):**
    `http://localhost:8080/`
    *Ответ: `server is running...`*

*   **Открыть книгу (с URL-кодированием):**
    *   Команда: `open_workbook --filepath "C:\My Docs\report.xlsx"`
    *   Запрос: `http://localhost:8080/?command=open_workbook%20--filepath%20%22C%3A%5CMy%20Docs%5Creport.xlsx%22`

*   **Записать значение в ячейку:**
    *   Команда: `set_cell --sheet_name "Лист1" --cell "A1" --value "Test Value"`
    *   Запрос: `http://localhost:8080/?command=set_cell%20--sheet_name%20%22%D0%9B%D0%B8%D1%81%D1%821%22%20--cell%20%22A1%22%20--value%20%22Test%20Value%22`

**Ответ сервера:**
*   В случае успешного выполнения команды сервер вернет текстовый ответ с результатом выполнения.
*   Пример ответа на `get_sheet_dimensions`: `Rows: 10, Cols: 5`

## Операции с Excel

### run_excel

Запускает приложение Excel. **Примечание:** Обычно не требуется, так как запуск `ExcelMcpPrj.exe` автоматически открывает Excel.

`run_excel`
Пример: `run_excel`

**Возможные ошибки:**
- `Excel already started`: Если Excel уже запущен.

### quit_excel

Закрывает приложение Excel. **Важно:** Эта команда не останавливает сервер `ExcelMcpPrj.exe`.

`quit_excel`
Пример: `quit_excel`

### stop_server

Останавливает сервер, завершая процесс `ExcelMcpPrj.exe` и закрывает Excel.

`stop_server`
Пример: `stop_server`

### ping_server

Проверяет, запущен ли сервер и отвечает ли он.

`ping_server`
Пример: `ping_server`

- Возвращает: `pong` если сервер активен.

## Операции с книгами

### create_workbook

Создает новую книгу Excel.

`create_workbook`
Пример: `create_workbook`

**Возможные ошибки:**
- `Error: Failed to create workbook`: Если не удалось создать книгу.

### open_workbook

Открывает существующую книгу Excel.

`open_workbook --filepath <путь>`

- `filepath`: Путь к файлу Excel
Пример: `open_workbook --filepath "C:\Users\User\Documents\test.xlsx"`

**Возможные ошибки:**
- `Error: Failed to open workbook: <filepath>`: Если не удалось открыть книгу по указанному пути.

### save_workbook

Сохраняет активную книгу Excel.

`save_workbook --filepath <путь>`

- `filepath`: Путь для сохранения
Пример: `save_workbook --filepath "C:\Users\User\Documents\test_saved.xlsx"`

**Возможные ошибки:**
- `Error: Failed save WorkBook, no active workbook`: Если нет активной книги для сохранения.
- `Error: Failed save, path not exists - <filepath>`: Если указанный путь для сохранения не существует.

### close_workbook

Закрывает активную книгу Excel.

`close_workbook`
Пример: `close_workbook`

**Возможные ошибки:**
- `Error: Failed to close workbook`: Если не удалось закрыть книгу.

## Операции с листами

### get_sheet_names

Получает список имен всех листов в активной книге Excel.

`get_sheet_names`
Пример: `get_sheet_names`

- Возвращает: Строка с именами листов, разделенными запятыми (например, "Лист1,Лист2,Лист3").

**Возможные ошибки:**
- `Error: Failed to get sheet names`: Если не удалось получить имена листов.

### get_sheet_dimensions

Получает размер используемого диапазона на листе (используя надежный метод `SpecialCells(xlCellTypeLastCell)`).

`get_sheet_dimensions --sheet_name <имя>`

- `sheet_name`: Имя листа
Пример: `get_sheet_dimensions --sheet_name "Лист1"`
- Возвращает: Количество строк и столбцов в формате "Rows: X, Cols: Y"

## Операции с ячейками

### set_cell

Записывает значение в ячейку. Не допустимо пытаться записать много ячяеек этой командой,
не проверяя результат в Command.out или Http(в зависимости от способа работы) 

`set_cell --sheet_name <имя> --cell <ячейка> --value <значение>`

- `sheet_name`: Имя листа
- `cell`: Адрес ячейки (например, A1)
- `value`: Записываемое значение
Пример: `set_cell --sheet_name "Лист1" --cell "A1" --value "Hello"`

**Возможные ошибки:**
- `Error: Failed to set cell, Workbook not opened`: Если книга не открыта.
- `Error: Failed to set sheet <sheet_name>`: Если указанный лист не существует.
- `Error: Failed to set cell <cell>`: Если указан неверный адрес ячейки.

### get_cell

Читает значение из одной ячейки.

`get_cell --sheet_name <имя> --cell <ячейка>`

- `sheet_name`: Имя листа
- `cell`: Адрес ячейки (например, A1)
Пример: `get_cell --sheet_name "Лист1" --cell "A1"`
- Возвращает: Значение ячейки

**Возможные ошибки:**
- `Error: Failed to get cell, Workbook not opened`: Если книга не открыта.
- `Error: Failed to get cell, sheet not exists <sheet_name>`: Если указанный лист не существует.
- `Error: Failed to get cell <cell>`: Если указан неверный адрес ячейки.

### read_range

Читает диапазон ячеек в виде текста. #13#10 - перенос строки; #9 - табуляция
Адреса ячеек следует определять с учетом спец символов

`read_range --sheet_name <имя> --start_cell <ячейка> --end_cell <ячейка>`

- `sheet_name`: Имя листа
- `start_cell`: Начальная ячейка диапазона (например, A1)
- `end_cell`: Конечная ячейка диапазона (например, D4)
Пример: `read_range --sheet_name "Лист1" --start_cell "A1" --end_cell "B5"`
- Возвращает: Текст, где столбцы разделены табуляцией (`	`), а строки - переводом строки (`
`). Пустые ячейки представлены пустыми строками между разделителями.

**Возможные ошибки:**
- `Error: Failed to set cell, Workbook not opened`: Если книга не открыта.
- `Error: Failed to read range, sheet not exists <sheet_name>`: Если указанный лист не существует.
- `Error: Failed to read range <start_cell>:<end_cell>`: Если указан неверный диапазон.

### get_cell_color

Читает цвет фона ячейки.

`get_cell_color --sheet_name <имя> --cell <ячейка>`

- `sheet_name`: Имя листа
- `cell`: Адрес ячейки (например, A1)
Пример: `get_cell_color --sheet_name "Лист1" --cell "A1"`
- Возвращает: Числовое значение цвета (integer).

**Возможные ошибки:**
- `Error: Failed to get cell color, Workbook not opened`: Если книга не открыта.
- `Error: Failed to get cell color, sheet not exists <sheet_name>`: Если указанный лист не существует.
- `Error: Failed to get cell color, invalid cell <cell>`: Если указан неверный адрес ячейки.

### set_cell_color

Устанавливает цвет фона ячейки.

`set_cell_color --sheet_name <имя> --cell <ячейка> --color <цвет>`

- `sheet_name`: Имя листа
- `cell`: Адрес ячейки (например, A1)
- `color`: Числовое значение цвета (integer). Например, 255 для красного, 65280 для зеленого.
Пример: `set_cell_color --sheet_name "Лист1" --cell "A1" --color 255`

**Возможные ошибки:**
- `Error: Failed to set cell color, Workbook not opened`: Если книга не открыта.
- `Error: Failed to set cell color, sheet not exists <sheet_name>`: Если указанный лист не существует.
- `Error: Failed to set cell color, invalid cell <cell>`: Если указан неверный адрес ячейки.

## Поиск информации

### find_in_workbook

Выполняет поиск указанного текста по всем рабочим листам активной книги Excel.

`find_in_workbook --text <текст_для_поиска>`

- `text`: Текст или часть текста для поиска.
Пример: `find_in_workbook --text "some text"`
- Возвращает: Строка, содержащая список найденных совпадений. Каждое совпадение представлено в формате "ИмяЛиста,АдресЯчейки,ЗначениеЯчейки" и разделено переводом строки (`
`). Если совпадений нет, возвращается пустая строка.

**Возможные ошибки:**
- `Error finding in workbook: <сообщение_об_ошибке>`: Если во время поиска произошла ошибка.
