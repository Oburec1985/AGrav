Есть вариант ручной установки через Github
[Ссылка на GitHub Dataview](https://github.com/blacksmithgu/obsidian-dataview)

**Инструкция по установке на Github**:
### **Шаг 1: Найдите плагин на GitHub**
1. Перейдите на [GitHub](https://github.com/).
2. В поисковой строке (вверху) введите название плагина, например: **obsidian-dataview**.
### **Шаг 2: Скачайте плагин**
1. На странице плагина найдите кнопку **Code** (зеленая кнопка справа). Нажмите на неё → выберите **Download ZIP**.
2. Файл скачается в папку «Загрузки» (или ту, которую вы указали).
### **Шаг 3: Распакуйте архив**

### **Шаг 4: Установите плагин в Obsidian**

1. Откройте папку вашего хранилища Obsidian (где хранятся заметки).
    
2. Перейдите в скрытую папку `.obsidian` → `plugins`. Если её нет — создайте.  
    **Важно!** Убедитесь, что видите скрытые файлы
    
3. В папку `plugins` скопируйте **распакованную папку плагина** (например, `obsidian-dataview-main`).
    
4.  Переименуйте в **правильное название**, как в официальном списке плагинов для Obsidian  (например, `dataview`, а не `obsidian-dataview-main`).
### **Шаг 5: Активируйте плагин**

1. Перезайдите в Obsidian.
    
2. Перейдите в **Настройки** → **Community plugins**.
    
3. Нажмите **Обновить** (круговая стрелка), если плагин не появился.
    
4. Найдите плагин в списке (например, Dataview) → включите <span style="color:rgb(0, 0, 0)">тумблер</span>.

### **Если плагин не появился:**

- Убедитесь, что папка плагина имеет **правильное название** (например, `dataview`, а не `obsidian-dataview-main`).
    
- Перезапустите Obsidian.
    
- Проверьте, нет ли ошибок в консоли (Настройки → Console).

- Некоторые плагины требуют **дополнительных настроек** — читайте их документацию (обычно файл `README.md` в репозитории).
